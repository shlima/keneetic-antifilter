# Комментарии (rem) для маршрутов в .bat-файлах

## Цель

Добавлять к каждой строке маршрута комментарий с именем сервиса, чтобы при
импорте в Keenetic у статического маршрута было описание. Формат строки:

```
route ADD 91.108.4.0 MASK 255.255.252.0 0.0.0.0 :: rem telegram
```

где `telegram` — имя сервиса.

## Источник комментария

Имя сервиса выводится из имени выходного файла: basename без суффикса
`-ipv4.bat`.

- `routes/telegram-ipv4.bat` → `telegram`
- `routes/cloudflare-ipv4.bat` → `cloudflare`

Если суффикс `-ipv4.bat` отсутствует — берётся basename без расширения `.bat`
(запасной вариант).

## Почему не мапинг по IP для all-ipv4-*.bat

Файлы `all-ipv4-*.bat` создаются целью `slice-routes` в Makefile простой
построчной конкатенацией всех `routes/*-ipv4.bat` (`cat ... | split -l 1024`).
Поскольку комментарий вшивается в каждую строку посервисного файла на этапе
генерации, при нарезке он переносится автоматически — каждая строка несёт свой
`:: rem`. Отдельный проход с сопоставлением по IP не нужен; дубликаты IP между
сервисами не создают неоднозначности.

## Изменения в коде

1. `internal/pkg/convertion/convertor.go`: добавить поле `comment string` в
   `Convertor` и параметр в `New(...)`. В формируемую строку дописывать
   ` :: rem <comment>`, если comment не пустой.
2. `cmd/main.go`: вычислять имя сервиса из `outputPath` (basename без
   `-ipv4.bat`) и передавать в `convertion.New(...)`.

Makefile и команды запуска не меняются — имя сервиса уже содержится в выходном
пути.

## Тесты

- `convertor`: строка с непустым comment получает суффикс ` :: rem <comment>`;
  с пустым comment — без суффикса (обратная совместимость).
- Вывод имени сервиса из пути: `routes/telegram-ipv4.bat` → `telegram`;
  путь без `-ipv4.bat` → basename без `.bat`.
